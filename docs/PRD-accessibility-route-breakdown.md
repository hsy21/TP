# PRD: 루트 상세 - 장애인 이동 지수 감점 분석 & 미니 지도

- **문서 상태:** Draft
- **작성일:** 2026-06-05
- **관련 화면:** 커뮤니티 피드(추천 루트 / 최근 등록 루트) → 루트 상세 모달
- **관련 코드:** `web/script.js` (`calculateAccessScore` L1297, `openCommunityPlanModal` L1581, 지도 L521·L570), `server/server.js` (`accessibility_reviews` L91, `community_plan_items` L157)

---

## 1. 배경 / 문제 정의

커뮤니티 피드의 추천 루트·최근 등록 루트 카드에는 **장애인 이동 지수(accessScore)** 가 배지와 진행률 바로 표시된다. 그러나:

1. **점수 산출 근거가 불투명하다.** 현재 `calculateAccessScore()`는
   `50 + (평점 × 5) + (설명에 '휠체어' 포함 시 +10) + (제목에 '배리어프리' 포함 시 +15)`
   라는 키워드/평점 기반 추정치다. 실제 동선의 경사·턱·환승 같은 물리적 장벽을 반영하지 못한다.
2. **사용자가 "왜 이 점수인지" 알 수 없다.** 루트를 눌러 상세 모달을 열어도 장소 목록과 후기 버튼만 있고, **어느 구간에서 왜 지수가 깎였는지** 보여주지 않는다.
3. 휠체어·유아차·노약자 이용자가 루트 채택(이어 붙이기/덮어쓰기) 전에 위험 구간을 미리 파악할 방법이 없다.

**핵심 사용자 요구:** "루트를 눌렀을 때, 작은 지도와 함께 **이동 지수가 깎인 구간과 이유**를 보고 싶다."

## 2. 목표 (Goals)

- 루트 상세 모달에서 **구간 단위(장소→장소)로 이동 지수 감점 항목과 사유를 시각화**한다.
- 모달 안에 **미니 지도**를 넣어, 감점이 큰 구간을 지도 위에서 색상으로 강조한다.
- 이동 지수를 **설명 가능한(투명한) 감점 모델**로 재정의한다 (총점 = 100 − Σ감점).

### Non-Goals (이번 범위 제외)

- 외부 보행 접근성 API(공공데이터 등) 실시간 연동 — 추후 과제.
- 실내 길안내, 턴바이턴 내비게이션.
- 사용자 장애 유형별 완전 개인화 점수(프로필 `mobility_type` 가중치는 v2에서 검토).

## 3. 대상 사용자 / 사용 시나리오

- 휠체어(수동/전동) 이용자, 유아차 동반자, 노약자 및 동행 보호자.
- **시나리오:** 피드에서 "지수 72점" 루트를 보고 클릭 → 상세 모달의 미니 지도에서 2번째→3번째 구간이 빨갛게 표시됨 → 감점 목록에서 "지하철 환승 엘리베이터 없음 −12, 보도 경사 급함 −6" 확인 → 위험 구간을 인지하고 채택 여부 결정.

## 4. 기능 요구사항

### 4.1 이동 지수 감점 모델 (설명 가능 점수)

기존 단일 숫자 추정을 **구조화된 감점 합산**으로 대체한다.

```
accessScore = max(0, 100 − Σ(구간별·장소별 감점))
```

각 감점 항목은 다음 필드를 가진다:
- `segmentIndex`: 어느 구간/장소인지 (예: 2 → "②번 장소" 또는 "②→③ 구간")
- `category`: 감점 유형 (아래 표)
- `penalty`: 차감 점수(양수)
- `reason`: 사용자에게 보여줄 한 줄 설명

**감점 카테고리(초기안):**

| category | 데이터 출처 | 예시 사유 | 기본 감점 |
|---|---|---|---|
| `step` (턱/단차) | `accessibility_reviews.door_step` | "입구에 10cm 이상 턱" | door_step 등급별 0~15 |
| `ramp` (급경사) | `accessibility_reviews.ramp_angle` | "경사로 각도 급함" | 0~12 |
| `narrow` (좁은 통로) | `accessibility_reviews.space_width` | "휠체어 통과 폭 부족" | 0~10 |
| `no_elevator` (환승/수직이동) | `access_tags`에 엘리베이터 태그 부재 | "지하철 환승 시 엘리베이터 없음" | 12 |
| `transport` (이동수단) | `community_plan_items.transport`/거리 | "도보 구간 1km 초과·경사" | 거리·수단 기반 0~8 |
| `no_data` (정보 부족) | 후기 없음 | "접근성 후기가 없어 확인 불가" | 5 (불확실성 패널티) |

> 감점 가중치는 설정 객체(`ACCESS_PENALTY_WEIGHTS`)로 분리해 추후 튜닝 가능하게 한다.

### 4.2 감점 산출 위치 (서버 vs 클라이언트)

- **권장:** 서버에서 산출하여 일관성 확보. 신규 엔드포인트
  `GET /api/community/:id/access-breakdown`
  반환 예:
  ```json
  {
    "score": 72,
    "segments": [
      { "index": 1, "name": "서울역", "deductions": [] },
      { "index": 2, "name": "남산타워", "deductions": [
        { "category": "no_elevator", "penalty": 12, "reason": "환승역 엘리베이터 없음" },
        { "category": "ramp", "penalty": 6, "reason": "진입 경사로 각도 급함" }
      ]}
    ],
    "totalDeducted": 28
  }
  ```
- 클라이언트는 이 응답으로 미니 지도 강조와 감점 목록을 렌더링한다.
- (대안: 1차로 클라이언트에서 기존 `plan.items` + 후기 데이터로 계산하고, 서버 이전은 후속.)

### 4.3 루트 상세 모달 UI

`openCommunityPlanModal`(`web/script.js:1581`) 및 `#communityPlanModal`(`web/index.html`)에 다음을 추가:

1. **미니 지도 영역** (`#cpModalMap`)
   - 기존 Leaflet/OSM·OSRM 자산 재사용(`web/script.js` L521·L570).
   - 장소 마커 + 경로 폴리라인 표시.
   - **감점 큰 구간은 색상 강조:** 감점 0 = 초록, 1~9 = 주황, 10+ = 빨강 폴리라인.
   - 강조 구간 클릭/탭 시 해당 감점 사유 툴팁 표시.
   - 모달 표시 직후 `map.invalidateSize()` 호출(기존 패턴 L398·L440 참고).

2. **감점 내역 패널** (`#cpModalBreakdown`)
   - 상단: "이동 지수 72점 — 28점 감점됨" 요약.
   - 구간별 리스트: `②번 남산타워` 아래에 감점 항목(아이콘 + 사유 + `−12`).
   - 감점 없는 구간은 "장벽 없음 ✓" 으로 간결 표시.
   - `no_data` 항목은 회색으로 "정보 부족" 안내 + 후기 작성 유도(기존 후기 버튼 연계).

3. 기존 장소 목록/후기/액션 버튼(이어붙이기·덮어쓰기·평가) 레이아웃은 유지하고 그 위/사이에 배치.

### 4.4 피드 카드 일관성

- 카드 배지(`지수 N점`)와 진행률 바는 새 모델의 `score`를 그대로 사용한다.
- 점수 색상 임계값(80/60)은 기존(`web/script.js:1390`) 유지.

## 5. 데이터 / API 영향

- **신규:** `GET /api/community/:id/access-breakdown` (서버 산출안 채택 시).
- **활용 기존 테이블:** `accessibility_reviews`(door_step·ramp_angle·space_width·access_tags), `community_plan_items`(좌표·순서·transport).
- **스키마 변경 없음** (초기). 감점 가중치는 코드 상수로 관리.
- 좌표가 없는 장소는 지도 강조에서 제외하고 목록 감점만 표시.

## 6. UX / 엣지 케이스

- 장소 1개뿐인 루트 → 구간 없음, 장소 단위 감점만 표시.
- 후기가 전혀 없는 루트 → 전 구간 `no_data`, "정보 부족으로 추정 점수입니다" 배너.
- 좌표 누락/지도 로드 실패 → 미니 지도 숨기고 감점 목록만 표시(우아한 저하).
- 모바일: 지도 높이 축소(예: 180px), 감점 목록은 세로 스택.

## 7. 성공 지표 (Success Metrics)

- 루트 상세 모달 열람 후 **감점 패널 노출률** 95%+ (오류로 미표시 < 5%).
- 상세 모달 → 루트 채택(이어붙이기/덮어쓰기) 전환율 개선.
- "점수 근거를 알 수 있다"에 대한 사용자 피드백 정성 평가.

## 8. 단계별 구현 (제안)

1. **P1 — 점수 모델 리팩터링:** `calculateAccessScore`를 감점 합산 구조로 교체(클라이언트), 구간별 `deductions` 산출.
2. **P2 — 모달 미니 지도:** 기존 지도 코드 재사용해 `#cpModalMap` 렌더 + 구간 색상 강조.
3. **P3 — 감점 패널 UI:** 구간별 사유 리스트 + 요약.
4. **P4 — 서버 이전:** `/access-breakdown` 엔드포인트로 산출 일원화, 가중치 외부화.
5. **P5 (옵션) — 개인화:** `user_profiles.mobility_type` 기반 가중치 조정.
